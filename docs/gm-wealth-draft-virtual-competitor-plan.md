# 个人财富、汇票、虚出与竞品系统开发计划

## 开发原则

- 先接资金闭环，再接风险闭环，最后接外部竞争压力。
- 每阶段都保持游戏可运行、旧存档可加载，并把关键数值暴露到界面。
- 现有 `src/App.jsx` 是单文件 React 状态机，本轮继续沿用本地 `useState` 模式，避免引入新框架或大规模拆分。

## 阶段 1：总经理财富与汇票采购地基

目标：让玩家能看到个人账户、调整垫资，并用银行承兑汇票采购车辆。

- 新增 `gmWealth` 状态：个人账户、月薪、收入、垫资、士气、历史记录。
- 新增 `drafts` 状态：未到期汇票、授信、银行信用、手续费和兑付统计。
- 改造订车付款方式：现金、3个月汇票、6个月汇票。
- 汇票采购只扣保证金和手续费，记录到期日和授信占用。
- 每日流程检查到期汇票：现金足够自动兑付，不足则逾期、扣信用和士气。
- 新增“总经理”Tab 展示财富、垫资、汇票概览和到期预警。
- 存档/读档/自动存档补齐新字段。

## 阶段 2：薪资、分红、垫资闭环

目标：把 GM 个人经济层接进月度经营循环。

- 月初自动发放 GM 月薪并计入人工成本。
- 总经理办公室支持薪资调整，处理最低工资、加薪条件和连续降薪士气事件。
- 现金紧张和汇票兑付不足时引导个人垫资。
- 公司现金恢复后自动归还垫资。
- 年底根据投资人评分与年度净利润结算分红。
- 接入个人财富成就和 GM 士气对团队效率的轻量影响。

当前进度：已完成月初发薪、现金恢复后的垫资自动归还、年度分红计算、年度净利润和投资人评分追踪；成就和士气对团队效率的深度影响留到阶段 6 调优。

## 阶段 3：月底虚出系统

目标：形成“冲销量拿返利，但埋下汇票雷”的核心冒险玩法。

- 月底 25-28 日开放“月底冲刺”面板。
- 从库存选择虚出车型和数量，计入账面销量与返利池。
- 虚出车辆与汇票关联，移出可售库存但保留浮库追踪。
- 真实销售优先消化同车型虚出车，避免重复计销量和返利。
- 月底厂家抽查、怀疑度、处罚、浮库资金占用费上线。

当前进度：已完成虚出状态、D25-D28月底冲刺面板、库存虚出、账面销量和返利入池、厂家怀疑度、浮库成本、厂家抽查罚款，以及后续真实销售优先消化虚出车。

## 阶段 4：竞品市场与反制措施

目标：让玩家面对外部抢客、价格战和本品店关系。

- 开局增加市场规模选择，生成竞品门店列表。
- 每月驱动竞品降价、活动、直播、挖人等行为。
- 新增“市场”Tab：市场份额、竞品卡片、情报时间线、反制措施。
- 市场份额影响返利、任务压力和退网风险。
- 本品店关系、联合抗竞、厂家干预和价格战规则上线。

当前进度：已完成开局市场规模选择、竞品门店生成、竞品市场Tab、市场份额展示、竞品月度行动、基础情报时间线、客流/转化分流压力，以及跟价降价、差异化服务、老客转介绍、联合抗竞四个反制措施。

## 阶段 5：三张财务报表改造

目标：让利润、现金流和隐藏负债都清晰可见。

- 在现有财务区新增利润表、现金流量表、资产负债表、趋势图切换。
- 单列汇票保证金、手续费、到期兑付、逾期罚息。
- 单列虚出收入、成本、返利、罚款和浮库成本。
- 投资人月审接入现金覆盖天数、资产负债率、逾期汇票和厂家罚款。
- 最近 12 个月历史报表用于趋势图和年底分红。

当前进度：已在“衍生与财务”加入利润表、现金流量表、资产负债表和趋势图切换；现金流单列汇票现金支出、GM垫资和归还；资产负债表显性展示应付汇票、GM垫资、可售库存与虚出/浮库库存。

## 阶段 6：数值调优与验收

目标：让四大系统形成稳定、可解释、可玩且不崩的经营循环。

- 校准汇票手续费、返利、库存周转、逾期惩罚和竞品抢客强度。
- 验证旧存档、新开局、自动存档、破产、胜利、解聘分支。
- 补齐关键 UI 提示，确保玩家知道风险来自哪里。
- 执行 lint/build，修复所有运行时和构建问题。

当前进度：已完成信息架构收口优化，包括开局市场环境合并、经营区域与市场规模绑定、投资人权重说明、消息中心降噪、月底冲刺迁移至销售运营、财务/返利/汇票合并为同一资金驾驶舱。已完成第一轮轻量数值校准：降低竞品抢客压力上限并放缓竞品压力增长，避免新手局过早被竞品分流压死；完整经济平衡仍需要 3-6 个月长局测试后继续微调。
