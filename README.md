# 奥迪 4S 店经营模拟

一个基于 React + Vite 的奥迪 4S 店经营模拟游戏。玩家扮演受投资人、厂家、银行和本地市场共同约束的运营总经理，在现金流、库存、销量、CSI、人员稳定、返利和融资风险之间做取舍。

## 游戏特色

- **剧本目标**：支持“自由模式”“6个月保住职位”“12个月净资产翻倍”“12个月区域明星店”等局内目标。
- **难度模式**：支持新手、标准、硬核三档难度，影响开局现金、库存融资授信、厂家任务和 CSI 起点。
- **新手引导**：引导玩家完成第一次订车、到货、布展、成交和月结。
- **完整经营循环**：订车、采购目标、到货、布展、投流、邀约、成交、返利、月结、投资人评价。
- **销售与客户谈判**：客户会根据同城报价、服务能力、金融方案、销售能力和 CSI 决定是否成交。
- **销售机会池**：日结会把线索、试驾、报价、客户回访和流失风险汇总为可操作机会，玩家可筛选、邀约、报价、跟进或关闭机会。
- **客户中心/CRM**：重点客户成交、战败或放弃后会沉淀为客户档案，日结生成转介绍、复活、满意回访和续保跟进机会。
- **财务与报表**：包含利润表、现金流量表、资产负债表、趋势图、月报和年报中心。
- **融资系统**：区分库存融资授信和承兑汇票专项授信，支持汇票到期兑付、逾期罚息和银行信用。
- **利润中心**：总经理办公室、财务报表、返利、汇票、GM 个人垫资和投资人授权。
- **本地市场**：同城竞品、价格战、本品店关系、市场份额、情报可信度、客户战败复盘、竞品活动和反制措施。
- **经营事件**：突发风险和机会会进入事件中心，玩家可选择处理方案，成本、成功率和逾期压力随难度变化。
- **衍生业务**：金融佣金、保险、精品、延保、续保、二手车、售后服务和 CSI 联动。
- **组织人事**：DCC、销售、客服、主播、售后技师的招聘、培训、标签、忠诚度、压力和流失风险。
- **结局系统**：破产、解聘、续约、优秀总经理、区域明星店等结果。
- **存档系统**：支持自动存档、多个手动存档槽、读档、重命名和旧存档兼容。

## 游戏指导手册

### 开局选择

进入游戏后先配置 4 个关键条件：

- **经营区域**：决定市场需求、获客成本、人才流失和授信倍率。
- **投资人**：决定月底评分偏好，例如现金安全、销量、利润、CSI 或成长性。
- **剧本目标**：自由模式不限期；目标剧本会在固定月份检查职位、净资产或区域表现。
- **难度模式**：新手、标准、硬核会影响初始现金、库存融资授信、厂家任务和 CSI 起点。

建议新玩家先用“标准难度 + 自由模式”熟悉经营循环；想要明确胜负压力时再选择 6 个月或 12 个月剧本。

### 每天的经营顺序

推荐按这个顺序操作：

1. **看首页待办**：先处理审批中心、经营事件、收件箱和高风险提醒。
2. **查现金和库存**：确认自有现金、库存融资负债、展厅车、库房车和在途车。
3. **订货或调展**：在“厂家订货”补充未来库存，在“展厅定价”布展、调价、设试驾车或处理老库存。
4. **投放营销**：在“漏斗营销”设置日预算或发起活动，保证 DCC 和销售有线索可转化。
5. **推进客户**：在“机会池”“客户谈判”“客户中心”处理邀约、报价、成交、回访和续保机会。
6. **盯住组织能力**：在人事和售后页补人、培训、留任，避免销售、DCC、客服或技师成为瓶颈。
7. **进入下一天**：点击“进入下一天”结算线索、客流、销量、成本、事件、竞品和员工状态。

月底前重点看“返利”“财务”“汇票”和“月底冲刺”。D25-D28 会开放冲刺操作，但虚出和激进压库会带来后续风险。

### 核心模块说明

- **首页 / 经营看板**：查看经营日历、现金、融资、剧本进度、待办、审批和消息。
- **门店总览**：用楼层蓝图查看展厅、库房、DCC、财务交付、售后、二手车和员工工位。
- **事件中心**：处理突发风险或机会；逾期会自动结算，风险事件通常会造成负面影响。
- **展厅定价**：调整车型挂牌价，移动展厅/库房车辆，给老库存补贴，设置或退役试驾车。
- **厂家订货**：选择车型、颜色、数量和付款方式；现金、库存融资、3 个月汇票、6 个月汇票各有现金流代价。厂家分为总部与区域大区：总部管理返利、指导价和品牌纪律，大区管理采购目标、超采奖励和区域资源。
- **漏斗营销**：配置采买线索、直播、车展等活动，影响线索池、到店和成交机会。
- **机会池 / 客户谈判 / 客户中心**：承接日结产生的销售机会，完成报价、跟进、成交、战败复盘和老客运营；战败客户画像会反哺本地市场情报。
- **总经理办公室 / 财务 / 返利 / 汇票**：管理总经理薪酬、个人垫资、授权谈判、三张报表、厂家任务返利和票据风险。
- **人事招聘 / 设施升级**：招聘并培养 DCC、销售、客服、主播、售后技师，升级展位和库容。
- **本地市场 / CSI 满意度**：查看同城竞品、价格战、情报可信度和市场份额，通过关怀和回访维护满意度。
- **二手车 / 售后服务 / 金融衍生**：管理二手车展厅、整备零售、售后产能，以及精品、延保、保险续保策略。

### 生存建议

- 不要只追销量。低价成交会改善现金回笼，但可能压低毛利、返利质量和投资人评分。
- 现金覆盖天数低时，优先减少订货、处理老库存、谨慎投放活动，并关注汇票到期。
- 展厅没有车会影响自然客流，库房太满会拖累资金效率；布展和订货要留提前量。
- CSI 下滑会影响成交率和投诉风险，售后技师、人手压力和客户回访都要跟上。
- 员工压力过高或忠诚过低会带来流失，关键岗位建议培训和留任并用。
- 竞品价格战时，不一定只靠降价；情报可信度低时先做市场摸底，再通过市场反制、金融方案、售后和衍生利润补回来。

### 存档与读档

游戏会在日结时自动存档，也支持 5 个手动存档槽。手动存档适合在月末、重大订货、融资或冲刺前保留决策分支。

存档保存在浏览器 `localStorage` 中；清空浏览器站点数据会删除本地存档。

## 技术栈

- React 19
- Vite 8
- Tailwind CSS 4
- ESLint

## 本地运行

```bash
npm install
npm run dev
```

```text

```

## 存档说明

游戏存档保存在浏览器 `localStorage` 中，包含自动存档和多个手动存档槽。清空浏览器站点数据会删除本地存档。

读档弹窗会展示自动存档和 5 个手动槽位。空槽位不可读取，有存档的槽位可读取、重命名或删除。

## GitHub 上传建议

上传源码即可，不需要上传：

- `node_modules/`
- `dist/`
- `backups/`
- `github-upload-*/`
- `.git/`
- `.DS_Store`
- `.env`

这些规则已经写入 `.gitignore`。GitHub 拉取后执行 `npm install` 即可恢复依赖。

## GitHub 发版步骤

发布前建议按下面顺序操作：

1. 确认工作区只包含要发布的源码、文档和脚本改动。
2. 运行完整验证：

```bash
npm run lint
npm test
npm run build
npm run test:ui
```

3. 提交代码：

```bash
git add .gitignore AGENTS.md README.md docs scripts src package.json package-lock.json eslint.config.js index.html public vite.config.js
git commit -m "Release dealership simulation gameplay update"
```

4. 推送到 GitHub：

```bash
git push origin HEAD
```

5. 在 GitHub 创建 Release 或 PR，发布说明建议包含：

- 架构拆分：核心配置、状态、纯经营逻辑、应用编排和 UI 模块已拆分到独立目录。
- 经营增强：客户中心、机会池、经营事件、竞品情报、厂家采购目标和厂家总部/区域双角色。
- 验证结果：`lint`、经营 smoke、生产构建和 UI smoke 已通过。
