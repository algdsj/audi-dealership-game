# Audi 4S 店经营模拟

一个基于 React + Vite 的奥迪 4S 店经营模拟游戏。玩家扮演的不是老板，而是受投资人约束的运营总经理，需要在现金流、库存、销量、CSI、人员稳定和厂家/银行/投资人关系之间做取舍。

## 核心玩法

- **区域开局**：选择低竞争三四线、中竞争省会、高竞争一线或新能源强势区，不同区域会影响客流、价格、获客成本、授信、人才流失和置换机会。
- **投资人压力**：不同投资人关注不同指标，包括销量增长、净利润、现金安全、库存周转、CSI 和人员稳定。
- **销售运营**：订车、定价、展厅布展、试驾车、二网批售、厂家返利和月底考核。
- **客户漏斗**：线上投流、DCC 邀约、自然进店、销售转化、价格审批和客户投诉。
- **二手车业务**：置换收购、整备、定价、零售、批售和强制处置。
- **售后与续保**：售后维修、技师能力、续保佣金和 CSI 联动。
- **外部谈判事件**：向厂家争取补贴/营销支持，向银行申请临时授信，向投资人申请追加现金或解释亏损。
- **员工标签**：DCC、销售顾问和技师有不同性格/标签，影响线索处理、转化、金融佣金、客诉和流失风险。
- **月底评价**：投资人每月评价经营结果，可能追加授权、限制订车、要求降本，连续差评会导致被解聘。

## 本地运行

```bash
npm install
npm run dev
```

打开终端输出的本地地址，通常是：

```text
http://127.0.0.1:5173/
```

## 常用命令

```bash
npm run dev      # 启动开发服务器
npm run build    # 构建生产版本
npm run lint     # 运行 ESLint 检查
npm run preview  # 预览构建结果
```

## 项目结构

```text
src/
  App.jsx        # 主要游戏逻辑和界面
  main.jsx       # React 入口
  index.css      # Tailwind CSS 入口
public/
  favicon.svg
  icons.svg
```

## 存档说明

游戏存档保存在浏览器 `localStorage` 中，包含自动存档和多个手动存档槽。清空浏览器站点数据会删除本地存档。
